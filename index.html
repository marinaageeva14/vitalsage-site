Beyond the browser demo

Want more than a snapshot? Run VitalSage from GitHub.

Comprehensive CLI analysis

Capture live CDP traces with per-function flame charts, run synthetic matrices across 5 networks × 3 viewports, and get findings from all nine agents — plus optional AI suggestions from Claude, GPT or Gemini. 

# audit any live URL — zero config
vitalsage trace --url https://example.com \
  --runs 5 --full-report --output report.html

Autonomous fix loop

The full cycle: measure → audit → AI-generated source patches → re-measure to verify. VitalSage edits your HTML/CSS/JS, confirms the improvement against a baseline, and retries until metrics move or retries run out. 

vitalsage fix --url http://localhost:5173 \
  --source ./src --retries 5 \
  --ai-provider anthropic --ai-key $KEY

Under the hood

How VitalSage works

Every report — in this demo, the CLI, or production RUM — flows through the same pipeline.

1

Measure

A real Chromium browser (Playwright) loads the page under emulated network and device profiles, capturing Core Web Vitals — LCP, FCP, TTFB, CLS, INP — plus a CDP flame-graph trace of the main thread. In RUM mode, the browser SDK captures the same data from real user sessions with zero overhead tricks: just PerformanceObserver.

2

Analyze

Nine specialized agents run in parallel over the sessions. Each one is rule-based first — fast, deterministic and free — checking exact thresholds against your data: unsized above-fold images, render-blocking scripts, missing preloads, slow server responses, long-task pileups and more.

3

Suggest

Findings become prioritized suggestions with severity, estimated impact, effort and code examples. Configure an AI provider (Claude, GPT or Gemini) and each agent additionally cross-correlates the full trace with your page context for root-cause answers rules can't reach.

4

Fix & verify

The fix command (or Claude Code via the bundled MCP server) closes the loop: it reads your source files, applies precise patches, re-measures against the original baseline, and only declares success when metrics actually improve beyond measurement noise.

Questions

FAQ

How does the analysis on this page work?

GitHub Pages is a static host, so this demo can't launch VitalSage's Playwright engine server-side. Instead it fetches real lab data for your URL from the Google PageSpeed API, then applies VitalSage's rule-based thresholds and renders the result in the same report format the CLI produces. The report for your selected device baseline is a real lab measurement; the other network profiles are estimates derived from it and labeled as such. For real throttled, multi-run traces, use the CLI from the GitHub repo.

What metrics does VitalSage capture?

All Core Web Vitals — LCP, FCP, TTFB, CLS and INP — rated against Google CrUX thresholds, plus main-thread trace metrics: Total Blocking Time, long-task counts, JS execute/compile time, rendering time, forced layouts, style recalcs, DOM node counts, JS heap and (with --full-report) a per-function CPU flame chart.

Do I need an AI API key?

No. VitalSage works fully without an LLM — the nine agents are rule-based and produce data-driven findings on their own. An API key (Anthropic, OpenAI or Gemini, or any OpenAI-compatible endpoint) is optional and adds deeper, cross-correlated suggestions plus the autonomous fix loop.

How do I set up Real User Monitoring?

Install the vitalsage npm package, call init() with a storage adapter that posts interactions to your collection server (a reference Express + SQLite server ships in the repo), and query /api/audit for streaming analysis of real traffic. The README has copy-paste integrations for vanilla JS, React and Next.js.

What does the autonomous fix loop actually change?

vitalsage fix reads your source directory, asks the AI for precise search/replace patches targeting the top-priority finding, applies them, and re-measures the page against a pre-change baseline. Safety rails: max 5 patches per attempt, capped file reading, a required --retries budget, and an improvement threshold so measurement noise never counts as success.

Can AI agents like Claude Code use VitalSage?

Yes — the repo ships an MCP server (vitalsage-mcp) exposing six tools: measure, analyze, real-user data fetch, before/after comparison, full route audit and a DOM inspector. Point Claude Code at a URL and it will loop measure → analyze → edit code → verify on its own.

What do I need to run it locally?

Node.js 20–22 and pnpm ≥ 9. Clone the repo, pnpm install && pnpm build, and either use your installed Chrome/Edge (automatic fallback) or download the bundled Chromium. Pages behind login work too, via --browser-channel chrome with a persistent profile.

Is it free?

Yes — VitalSage is fully open source under the MIT license. The only optional cost is your own AI provider key if you enable AI-enhanced suggestions or the fix loop.
